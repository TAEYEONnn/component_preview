1 누락된 화면
누락

자격증 접수 안내 — 외부 링크 진입 화면

"사이트 방문하기" 클릭 후 연결 방식(인앱 웹뷰 vs. 외부 브라우저) 미설계. 인앱 웹뷰 선택 시 TopBar(닫기/뒤로가기) 포함 화면 별도 필요.

누락

기출문제 — Empty State 화면

특정 자격증 기출문제 미존재 시 빈 상태 화면 미설계. 플로우 다이어그램에도 해당 분기 누락.

누락

기출문제 풀기 — 중간 이탈 확인 화면

플로우에 이탈 케이스 언급 있으나 이탈 확인 모달 화면 미설계. 타이머 구조상 이탈 시 처리 방식 정의 필요.

누락

자격증 삭제 — 확인 모달

삭제하기 버튼 설계됨. 삭제 전 확인 모달 미설계로 탭 즉시 삭제 실행됨.

누락

채점 결과 — 재응시 플로우

결과 화면 이후 재응시 CTA 및 연결 플로우 미정의. "해설 보기" 외 다음 액션 없음.

누락

비로그인 → 로그인 완료 후 복귀 플로우

비로그인 → 로그인 유도 바텀시트 케이스 있으나, 로그인 완료 후 복귀 화면 미정의로 플로우 단절.

2 UX 개선 제안
UX

문제 네비게이션 — 진행 바 + 문항 점프 제안

이전/다음 버튼만 제공, 특정 문항 점프 방법 없음. 진행 바 및 문항 인덱스 점프 기능 추가 제안.

제안 : 상단에 ProgressBar(현재 문항 / 전체) 노출. 탭 시 문항 목록 오버레이 표시.
UX

자격증 접수 안내 — 상태 배지 미분리

목록 아이템에 접수 상태 시각 구분 없음. 마감 자격증에도 버튼 동일 노출되어 혼란 발생 가능.

제안 : 기존 StatusBadge에 접수중 / 접수예정 / 접수마감 타입 추가. 마감 상태에서는 버튼 비활성 처리.
UX

자격증 선택 모달 — 미선택 시 버튼 제어 미정의

선택 안내 문구 있으나, 미선택 시 확인 버튼 처리(비활성/토스트) 설계서에 미명시.

제안 : 미선택 시 확인 버튼 비활성(disabled) 처리. 선택 즉시 버튼 활성화.
UX

기출문제 목록 — 응시 이력에 따른 버튼 분기 불명확

버튼 1개(full width)와 2개 구조 혼재. 분기 조건 설계서에 미명시.

제안 : 최초 응시 → "기출문제 풀기" 1개(full width). 응시 이력 있음 → "기출문제 풀기" + "해설 보기" 2버튼으로 명시.
UX

타이머 — 백그라운드 전환 시 동작 규칙 미정의

백그라운드 전환·화면 잠금·이탈 모달 노출 중 타이머 동작 정책 미정의. 시험 타이머는 정책 정의 필수.

제안 : 백그라운드 전환 시 타이머 계속 진행(카운트). 이탈 확인 모달 노출 시는 일시정지.
3 기획 제안 사항
디자인 진행 전 확정이 필요한 항목. 사전 결정 시 수정 리스크 최소화.
1

외부 사이트 연결 방식

인앱 웹뷰 vs. 외부 브라우저 확정 필요. 인앱 웹뷰 선택 시 TopBar 포함 화면 추가 설계.

2

자격증 접수 상태값 정의

접수중 / 접수예정 / 접수마감 상태별 버튼 노출 규칙 확정 필요.

3

기출문제 없을 때 Empty State 문구

기출문제 미존재 시 노출 문구 및 CTA 버튼 여부 확정 필요.

4

풀이 중 이탈 시 타이머 처리 정책

이탈 모달 노출 중 일시정지 여부, 백그라운드 전환 시 계속 진행 여부 확정 필요.

5

재응시 정책

재응시 허용 여부, 허용 시 횟수·시간 제한 정책 확정 필요.

6

비로그인 → 로그인 후 복귀 플로우

로그인 완료 후 기출문제 화면 직접 복귀(딥링크) 여부 확정 필요.

7

자격증 삭제 확인 모달 문구

삭제 확인 모달 노출 여부 및 문구 확정 필요.

8

문항 점프 네비게이션 허용 여부

풀이 중 특정 문항 점프 기능 허용 여부 확정 필요.
