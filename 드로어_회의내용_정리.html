아이콘 탭 방식 대신 드로어 메뉴 방식으로 최종 결정
메인 콘텐츠 볼륨이 커서 스크롤이 길어질 것으로 예상되는 상황에서, 아이콘 탭 방식보다 드로어가 사용성 측면에서 더 적합하다는 결론을 내렸다. 메뉴 자체 수가 많지 않아 수강생 입장에서 드로어로 전체 구조를 한눈에 보는 편이 낫다는 판단이 함께 작용했다. 단, 드로어 단독이 아닌 메인 화면과 드로어를 병행 배치하는 절충안으로 합의됐다.
논의 과정 드로어 vs 아이콘 탭 — 의견 충돌 및 합의 흐름
💡
제안 — 아이콘 탭 방식 (김태연)
드로어는 메뉴를 보려면 한 번 더 펼쳐야 하는 단계가 생기는데, 아이콘 탭 방식은 홈에서 모든 서비스가 바로 보여서 더 편리하다는 의견을 제시했다. 자주 쓰는 서비스가 아이콘으로 홈에 노출되면 탐색 없이 바로 진입이 가능하다는 점을 근거로 들었다.
🏗️
기획 판단 — 드로어 유지
메인 화면 콘텐츠가 많아 스크롤이 길어질 것으로 예상되어, 아이콘 탭이 홈 화면과 겹치거나 레이아웃을 잡아먹는 문제가 생길 수 있다고 판단했다. 메뉴 수가 적지 않아 수강생 기준에서 드로어로 전체 사이트맵을 한눈에 보는 편이 낫다는 기획 의도가 있었다.
🤝
합의 — 절충안 채택
아이콘 탭을 메인에 넣는 대신, 자주 쓰는 서비스(자스)를 메인과 드로어 양쪽에 동시 배치하는 방식으로 절충했다. 메인에서도 빠른 접근이 가능하고, 드로어에서는 전체 구조를 파악할 수 있는 두 가지 동선을 모두 제공한다.
🎨
UX 디자인 방향 — 배경 처리
드로어 오픈 시 배경이 살짝 보이는 방식보다 배경 블러 또는 딤 처리를 적용해 몰입감을 높이는 방향으로 의견이 모였다. 전체 화면을 덮는 UI 형태가 선호됐으며, 퍼블리싱 단에서 배경 블러 또는 반투명 오버레이 디자인 효과를 적용하기로 했다.

권한 처리 드로어 메뉴 진입 후 상태 분기 방식
⚙
동일 페이지 진입 후 권한에 따른 액션 분기
사용자 상태(게스트 / 비수강생 / 수강생)와 무관하게 드로어 메뉴 항목은 모두 동일한 페이지로 진입한다. 분기는 페이지 진입 이후, 권한에 따른 액션 처리 단계에서만 이루어진다. 즉, 같은 URL 구조를 유지하면서 서버가 반환하는 권한 정보에 따라 화면 내 요소 노출·잠금 여부가 결정되는 구조다.

단, 드로어 자체에서 노출되는 항목은 상태별로 달라지므로 드로어 컴포넌트는 사용자 상태를 참조해 렌더링해야 한다.
미결 액션 아이템 추가 기획 및 설계가 필요한 항목
게스트 / 비수강생 / 수강생별 드로어 메뉴 구성 상세 기획
전체 메뉴 항목 목록 나열 후, 각 항목마다 사용자 상태별 표시(노출) / 잠금 표시 / 숨김 여부를 표 형태로 정의해야 한다. 잠금 항목 클릭 시 처리 방식(Toast 안내 vs 로그인 화면 이동)도 함께 확정 필요.
